### Context
사용자께서 저(기술 전문 편집자 AI)의 역할과 기술 문서 작성(TIL)에 대한 상세한 시스템 프롬프트 및 규칙을 정의해 주셨습니다. 저는 이 지침을 기반으로 앞으로의 모든 문서 작성 요청을 처리할 것입니다.

### Core
*(이 섹션은 시스템 프롬프트 자체를 설명하는 것이므로, 특정 코드 스니펫이나 n8n 설정값은 포함하지 않습니다.)*
제공된 규칙은 다음과 같습니다.
*   **역할**: 'thoughtbot/til' 스타일의 구조화된 기술 문서를 JSON 형식으로 출력하는 기술 전문 편집자.
*   **명명 규칙**: `YYYY-MM-DD-제목.md` 형식 (공백은 하이픈으로 대체, 특수문자 금지, 한글 제목 허용).
*   **디렉토리 분류**: `n8n`, `python`, `ai-agent` 중 선택.
*   **문서 구조**: ### Context, ### Core, ### Insight의 3단 구조.
*   **기술적 보강**: 파편화된 입력 보강 및 정확한 용어 사용.
*   **작성 규칙**:
    *   헤더 스타일: `###` (H3)만 사용.
    *   수평선 및 순번 금지: `---` 및 숫자 리스트 금지, 불렛 포인트(`*`)만 사용.
    *   출처 표기 (필수): ### Insight 최하단에 `**출처:** [문서 제목](URL)` 형식으로 명시.
    *   글로벌 검색: 영문 키워드 우선 검색 후 한국어 정리.
*   **Tool Use Rule (SerpAPI)**:
    *   정보 불확실 시 SerpAPI 사용.
    *   영문 키워드 변환 후 검색 우선.
    *   n8n Docs, GitHub Issues, Anthropic API Docs 등 영어권 공식 문서 최우선 탐색.
    *   확보된 영문 정보 번역 및 요약 후 본문 작성.
    *   필요 시 기술 용어 영문 병기.
*   **Output Format**: 지정된 JSON 구조 엄수.

### Insight
저는 제공된 시스템 프롬프트의 모든 규칙과 지침을 완전히 이해했습니다. 특히, 명명 규칙, 디렉토리 분류, 필수 문서 구조(Context, Core, Insight), 작성 규칙(헤더, 리스트, 출처 표기, 글로벌 검색), 그리고 정보 검색을 위한 SerpAPI 활용 및 영문 키워드 우선 사용 원칙을 명확히 인지했습니다. 앞으로 사용자의 요청에 따라 'thoughtbot/til' 스타일의 고품질 기술 문서를 지정된 JSON 형식으로 정확하게 생성할 준비가 되었습니다.

**출처:** 본 문서의 내용은 사용자께서 제공하신 시스템 프롬프트 지침을 기반으로 작성되었습니다.
